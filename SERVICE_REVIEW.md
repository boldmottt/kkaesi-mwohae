## ✅ 강점

### 1. 디자인 시스템의 일관성 (높음)
- **일관된 컬러 팔레트**: `amber-400/500`을 primary로 전 페이지 통일 (login, onboarding, main, settings)
- **컴포넌트 재사용**: `WakeWindowSettings`가 온보딩과 설정 페이지에서 동일하게 사용됨 — DRY 원칙 준수
- **타이포그래피**: Inter (본문) + Gamja Flower (추억 페이지 제목)의 계층적 폰트 사용이 적절
- **모바일 최적화**: `max-w-md mx-auto` + `pb-16`(하단 네비 공간 확보)로 모바일 앱 같은 UX 구현

### 2. AI 기반 활동 추천의 과학적 근거 (높음)
- **OpenAI GPT-5.4-nano**를 활용한wake window별 활동 추천
- **CDC/WHO 근거 기반 프롬프트**: `lib/claude/activities.ts`에서 발달 단계별 적정 활동량, 주의집중시간을 명시적 프롬프트에 포함
- **고정 루틴 처리**: 루틴 위치(start/middle/end)를 파싱하여 AI 추천에서 정확히 제외 — 로직이 정교함
- **시간대별 컨텍스트**: `getTimeOfDayHint()`로 아침/오후/저녁에 맞는 활동 추천

### 3. 추억 페이지의 시각화 (매우 높음)
- **Dot Painting Canvas**: 날짜별 활동 수를 시드 기반 난수로 결정론적 시각화 — 재방문 시 동일한 패턴
- **Pentagon Radar Chart**: 5개 발달 영역(신체/감각/언어/인지/정서)의 균형 점수 시각화
- **Weekly Trend Bars**: 주간별 활동 카테고리별 막대 그래프
- **Cross-Domain Suggestions**: 가장 부족한 영역을 보완하는 크로스 도메인 활동 추천 (예: 신체+언어 = "동물 흉내 걷기")
- **AI 월간 코멘트**: `monthly-insight` API를 통한 전문가 스타일 코멘트

### 4. 사용자 인터랙션의 세밀함 (높음)
- **ActivityItem**: 완료 체크 → 표정 피드백(좋아함/보통/싫어함) → 메모 작성의 3단계 피드백 루프
- **실시간 저장**: 메모 입력 시 600ms debounce로 자동 저장 — UX 끊김 최소화
- **수면 카드**: 간밤 수면/낮잠/취침을 napIndex로 구분하여 시각적 피드백 (indigo-200 테두리)

## ⚠️ 개선 필요

### [중요도: 높음] 1. 하단 네비게이션에서 '이번 주 깨시' 페이지 누락
- **문제**: `BottomNav.tsx`에 `/week` 경로 링크가 없음. 네비게이션은 오늘/추억/설정 3개뿐
- **파일**: `src/components/ui/BottomNav.tsx` (L12-L38)
- **영향**: 주간 스케줄 확인 기능의 discoverability가 극히 낮음. 핵심 기능인데 진입 경로 없음
- **해결책**: BottomNav에 4번째 탭으로 "이번 주" 추가. 아이콘은 `📅` 또는 캘린더 SVG 권장

### [중요도: 높음] 2. 온보딩의 hardcoded 깨시 설정 (3회 고정)
- **문제**: `onboarding/page.tsx` (L12-L16)에서 깨시 횟수를 3회로 하드코딩. 실제 아기 나이에 따른 권장 깨시 시간이 없음
- **파일**: `src/app/onboarding/page.tsx` (L12-L16)
- **영향**: 0~3개월 아기(5~6회 깨시), 8+개월(2~3회 깨시) 등 발달 단계에 맞는 초기 설정이 필요
- **해결책**: `getDevStage()` 함수(이미 `lib/claude/infant-dev-reference.ts`에 존재)를 온보딩에서도 활용. 아기 생년월일 입력 시 권장 깨시 횟수/시간을 자동 채워넣는 기능 추가

### [중요도: 높음] 3. 주간 페이지의 기본 날짜가 '내일'
- **문제**: `week/page.tsx` (L10-L14)에서 `getDefaultDate()`가 오늘 + 1일을 반환. 사용자가 주간 페이지를 열면 내일 스케줄이 보임
- **파일**: `src/app/week/page.tsx` (L10-L14)
- **영향**: "이번 주"를 보려는 사용자에게 오늘/과거 데이터가 안 보여서 혼란. "오늘로" 버튼이 있지만 기본값이 잘못됨
- **해결책**: `getDefaultDate()`를 오늘 날짜로 변경. 또는 "오늘"을 첫 탭으로 포함

### [중요도: 높음] 4. API 인증 누락 (보안 취약점)
- **문제**: `sleep-logs`, `activity-logs`, `activities` 등 여러 API가 Supabase auth를 검증하지 않음. profileId만 받으면 데이터 조회/수정 가능
- **파일**: `src/app/api/sleep-logs/route.ts`, `src/app/api/activity-logs/route.ts`, `src/app/api/activities/route.ts`
- **영향**: 악의적인 사용자가 다른 사용자의 수면/활동 데이터를 조회하거나 수정 가능. 배우자 공유 기능과 충돌 가능성
- **해결책**: `chat` API처럼 `createClient()` → `auth.getUser()`로 인증 검증 추가. profileId가 현재 사용자의 profile과 일치하는지 확인

### [중요도: 높음] 5. 로그인 후 리다이렉트 실패 (매직 링크)
- **문제**: `login/page.tsx` (L18-L28). 매직 링크 로그인 성공 시 `/`로 리다이렉트하지 않고 `setSent(true)`만 함. Supabase auth callback에서 `/`로 이동해야 하는데, 그 경로가 명시적이지 않음
- **파일**: `src/app/login/page.tsx` (L18-L28)
- **영향**: 매직 링크 클릭 후 앱으로 돌아왔을 때 로그인 상태가 유지되지 않으면 무한 루프 발생
- **해결책**: `/auth/callback` 페이지 생성. Supabase auth 이벤트 리스너에서 `session` 감지 시 `/`로 자동 리다이렉트

### [중요도: 중급] 6. 에러 페이지 부재 (404, 500)
- **문제**: `error.tsx`, `not-found.tsx`, `loading.tsx`가 전역으로 존재하지 않음. Next.js 16에서 전역 에러 처리가 필요
- **파일**: `src/app/` 하위 전체
- **영향**: API 오류 시 "네트워크 오류가 발생했어요"만 표시. 사용자에게 명확한 에러 메시지 제공 안됨
- **해결책**: 전역 `src/app/error.tsx` 생성. `not-found.tsx`로 존재하지 않는 경로 처리. API 에러는 구체적인 메시지(예: "AI 추천 서비스 점검 중" 등)로 변경

### [중요도: 중급] 7. 추억 페이지의 다크 모드 스타일 불일치
- **문제**: `memories/page.tsx` (L172, L189)에서 `bg-gray-50 dark:bg-gray-900` 사용. 하지만 메인 페이지는 `bg-amber-50`. 다크 모드에서 추억 페이지가 완전히 다른 톤으로 보임
- **파일**: `src/app/memories/page.tsx` (L172, L189)
- **영향**: 페이지 간 시각적 일관성 깨짐. 메인 앱은 따뜻한 amber 톤인데 추억 페이지만 회색/검정
- **해결책**: `bg-amber-50 dark:bg-[#1a1420]` 등 amber 계열 다크 모드로 통일. `DayDetailView.tsx`의 `bg-[#FFF8F0]`도 amber-50으로 변경

### [중요도: 중급] 8. DailyChat의 메시지 비Persistence
- **문제**: `DailyChat.tsx` (L39-L61). 이전 대화 내역을 `daily_chat_sessions` 테이블에서 불러오지만, AI 응답 실패 시 하드코딩 에러 메시지만 표시. 대화 이력이 끊기면 UX가 단절됨
- **파일**: `src/components/daily-chat/DailyChat.tsx` (L101-L105)
- **영향**: AI 응답 실패 시 사용자가 다시 "오늘 하루 스케줄을 설계해줘"를 입력해야 함. 대화 컨텍스트 손실
- **해결책**: 실패 시 "다시 시도" 버튼 표시. 또는 캐시된 이전 세션의 마지막 메시지를 보여주고 "계속하기" 옵션 제공

### [중요도: 중급] 9. Activity 추천의 캐싱 전략 문제
- **문제**: `WakeWindowCard.tsx` (L66-L92). `hasFetched.current` 플래그로 최초 1회만 fetch. 하지만 수면 시간 변경 시 재fetch하지 않음 (start_time이 바뀌어도 무시)
- **파일**: `src/components/wake-window-card/WakeWindowCard.tsx` (L66-L92)
- **영향**: 사용자가 수면 시간을 수정하면 추천 활동이 outdated 상태가 됨. "다시 추천" 버튼은 있지만 자동 갱신 안됨
- **해결책**: `hasFetched` 대신 `lastStartTime`을 state로 관리. start_time 변경 시 재fetch 트리거

### [중요도: 낮음] 10. Invite 시스템의 만료 처리 부재
- **문제**: `invite/route.ts`에서 token 생성만 하고 만료일이 없음. `join/page.tsx`에서 `accepted_at` 체크는 있지만, 오래된 token이 무한히 유효
- **파일**: `src/app/api/invite/route.ts`, `src/app/join/page.tsx`
- **해결책**: invitations 테이블에 `expires_at` 컬럼 추가. 7일 만료 설정. 만료된 token은 join 페이지에서 "만료된 초대 링크" 메시지

## 💡 비즈니스 관점 제언

### 🔄 Retention (재방문율) 관련

1. **푸시 알림 기능 추가**
   - 깨시 시간 도달 시 "이제 깨시 시작이에요" 알림
   - 낮잠 시간 도달 시 "낮잠 시간입니다" 알림
   - 하루 활동 기록 독려 ("오늘 활동은 어떻게 되셨나요?")
   - **기대 효과**: DAU 30% 이상 증가, 하루 2회 이상 앱打开 유도

2. **성장 기록 타임라인**
   - 현재 추억 페이지는 월간/일일 단위지만, "아기 성장 기록"이라는 별도 뷰 추가
   - 첫 돌, 첫 뒤집기, 첫 걷기 등 마일스톤 자동 감지 (activity_logs의 rating=1 패턴 분석)
   - **기대 효과**: 공유 유도 → 바이럴 성장

3. **배우자 공유 기능 강화**
   - 현재 초대 링크는 있지만, 실시간 동기화가 안 보임. 두 배우자가 동시에 활동 기록할 때 충돌 처리 필요
   - "누가 언제 기록했는지" 표시 (activity_logs에 user_id 추가)

### 💰 Monetization (수익화) 관련

4. **프리미엄 기능 정의**
   - 현재 무료: 기본 활동 추천, 수면 기록, 추억 페이지
   - **유료화 후보**: 
     - AI 월간/주간 인사이트 (현재 무료이지만 AI 비용 발생)

     - 가족 공유 (현재 무료이지만 서버 비용 발생)
   - **제안**: "월간 인사이트"를 프리미엄으로 전환. 무료 사용자는 기본 데이터만, 프리미엄은 AI 코멘트 + 크로스 도메인 추천
